# Vital — анализ и план улучшений

## Архитектура (текущая)

| Режим | Хостинг | Данные | APK |
|-------|---------|--------|-----|
| **Dev / сервер** | `npm run dev` | SQLite + API | — |
| **Standalone** | GitHub Pages | localStorage на телефоне | Capacitor → URL Pages |

Standalone-сборка: `npm run build:pages` → папка `out/` → GitHub Actions → Pages.

APK — оболочка WebView (Capacitor), открывает тот же URL, что и браузер. Устанавливается с вашего сайта как `.apk`.

---

## Что уже улучшено

1. **Без индивидуализма по умолчанию** — нейтральный профиль, быстрый старт, нет предположений про диагнозы.
2. **Расширенные настройки** — цикл, лекарства, психология скрыты за кнопкой в Профиле.
3. **Локальное хранилище** — в standalone всё на устройстве, экспорт JSON.
4. **GitHub Pages CI** — автодеплой при push в `main`.
5. **PWA-иконки** — PNG 192/512.

## GitHub Pages + APK — как это работает

```
Push в GitHub
     ↓
Actions: build:pages → out/
     ↓
https://USER.github.io/REPO/
     ↓
Capacitor APK (WebView) ──→ тот же URL
     ↓
Пользователь: localStorage на телефоне
```

**Важно:** в настройках репозитория включите **Pages → Source: GitHub Actions**.

**Capacitor:** замените URL в `capacitor.config.json` на ваш реальный Pages-адрес, затем:

```bash
npm install @capacitor/core @capacitor/cli @capacitor/android
npx cap add android
npx cap open android
# Build → Build APK
```

APK выложите на сайт: `https://yoursite.com/vital.apk`.

## Ограничения standalone

- Нет синхронизации между телефоном и компьютером (только JSON-бэкап).
- Смена телефона — ручной экспорт/импорт (импорт — в roadmap).
- Первый запуск APK нужен интернет (загрузка с GitHub Pages); дальше UI может кэшироваться SW.
- GitHub Pages — публичный URL (приватный repo всё равно отдаёт статику по URL).

## Рекомендация

Для массовой аудитории оптимален **standalone + GitHub Pages + APK на сайте**: бесплатно, без сервера, без регистрации. Серверный режим (`npm run dev` / Vercel) оставьте для себя или для пользователей, которым нужна синхронизация.
